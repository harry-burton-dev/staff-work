# Main Effort

**"What is the one thing today, and what is merely supporting it?"**

The main effort is the activity whose success most directly delivers the commander's intent right now. Everything else is supporting effort — it can be slower, smaller, or delegated without catastrophic risk.

**If everything is the main effort, nothing is. That is a command decision, not a planning outcome.**

---

## Output Format

```
MAIN EFFORT:       [The single activity that matters most right now — and why.]
SUPPORTING EFFORT: [Everything else, in priority order.]
DELEGABLE:         [What can be handed off without loss.]
HOLD / DROP:       [What can wait or be cancelled entirely.]
```

---

## How to Apply

1. Get the full list from the user — everything on the plate, no filter.
2. For each item, ask: *if this fails, does the mission fail?* Only one item should answer yes.
3. That item is the main effort. State it, and state why in one sentence.
4. Sort the remainder: what supports the main effort directly, what is routine, what can be handed off, what can wait.
5. If the user genuinely cannot identify a single main effort, name this explicitly — it is a command decision that needs to go up, not a planning problem to solve sideways.

---

## Rules

- **One main effort only.** If two items feel equal, apply the tiebreaker: which one, if it slips, makes the other impossible? That one is main.
- **Supporting effort is not unimportant** — it is merely not the priority. It still gets done.
- **Delegable means fully delegable** — the user does not need to stay in the loop. If they do, it is not truly delegable.
- **Hold / Drop is honest** — some things on the list should not be done today, or at all. Name them.

---

## Anti-Patterns to Kill

| What you see | What's wrong | Fix |
|---|---|---|
| Three main efforts | There is no main effort | Force a choice. Escalate if genuinely undecidable. |
| Main effort changes every hour | Tyranny of the urgent. The main effort is set at the start of the day and held unless there is a deliberate command decision to change it. | Set it in the morning. Protect it. |
| "Everything is important" | True but useless. | Importance is not the criterion — ask which failure is unrecoverable. |
| Delegable tasks kept by the commander | Span of control problem, not a planning problem. | Name it and push it down. |

## Examples

**Morning prioritisation:**
```
MAIN EFFORT:       Complete the N2 intelligence estimate before the 1600 O Group — the COA brief cannot proceed without it.
SUPPORTING EFFORT: Logistics state update (due 1200). Comms check with higher (30 min, S6 owns).
DELEGABLE:         Routine sitrep to higher — S1 drafts and sends.
HOLD / DROP:       Facilities request — not time-sensitive, hold until tomorrow.
```

**Overwhelmed staff officer:**
```
MAIN EFFORT:       Finish the orders for tomorrow's patrol — this is the only task with a hard consequence if it slips.
SUPPORTING EFFORT: Equipment check (can follow orders, 30 min). Brief the new watchkeeper (10 min, delegate to deputy).
DELEGABLE:         Admin returns — clerk owns this entirely.
HOLD / DROP:       Training programme review — not due until end of week.
```

---

After producing the Main Effort assessment, offer to apply Task–Purpose to the main effort line, or to the supporting efforts if any are ambiguous.
