# Task–Purpose

**"Do X in order to Y."**

Every tasking gets a **what** and a **why**. The *in order to* is non-negotiable — it tells the recipient why, so they can improvise when the plan meets reality. It tells the sender whether they actually understand the task.

**If you cannot state the *in order to*, the task is not ready to be issued.**

---

## Output Format

```
TASK:    [Verb + object. The action. What must be done.]
PURPOSE: In order to [the outcome this task enables].
```

---

## How to Apply

1. Extract the core action from whatever the user gives you — email draft, verbal description, rough note, or full brief.
2. State it as an unambiguous verb + object. One action.
3. Ask: what does this task *enable*? That is the purpose.
4. Write the Task–Purpose pair.
5. Apply the three tests (below).
6. If converting to a written product (email, directive, order), write it — the Task–Purpose disciplines the content, but the output should match the register the user needs.

---

## Three Tests Before Issuing

- **Improvisation test:** Can the recipient, working only from this, make a sensible decision if conditions change?
- **Ownership test:** Does the purpose belong to *this* task, or to the higher operation? It should belong to *this* task.
- **Verb test:** Is the verb actionable and unambiguous? ("Ensure" and "support" fail this test.)

---

## Anti-Patterns to Kill

| What you see | What's wrong | Fix |
|---|---|---|
| "Ensure coordination with N4" | Not a task. No clear action, no outcome. | "Confirm fuel state with N4 by 1600 in order to complete the logistics annex." |
| "Support the operation" | Not a purpose — too high, too vague. | State what specifically becomes possible if this task succeeds. |
| "Liaise with the LO" | Liaise is not a verb. | "Brief the LO on the revised patrol route in order to confirm safe passage." |
| Purpose = restatement of the task | The why must be distinct from the what. | Ask: what is now possible because this task was done? |

---

## Examples

**Simple tasking:**
```
TASK:    Issue the updated reconnaissance tasking to the FRG by 1400.
PURPOSE: In order to ensure the patrol plan reflects current ground truth before the O Group at 1600.
```

**Email context:**
```
TASK:    Provide a consolidated equipment readiness state to the ops room by 0800 daily.
PURPOSE: In order to give the duty officer an accurate picture before the morning brief.
```

**Ambiguous request clarified:**
```
TASK:    Review the draft communications plan and return comments to N6 by COB Thursday.
PURPOSE: In order to identify gaps before the plan is staffed to higher headquarters Friday.
```
